# Hades Watch MMO Hub Expansion: Field Ops Underwatch

## Purpose

The Hades Watch MMO Hub, also known as **Field Ops**, is the central command layer for the Hades Watch web MMO. It is where an operative sees who they are, what they are building, which faction signals they carry, what missions are available, what ciphers remain unsolved, what community spaces are active, and what the Archive has recovered.

The expanded hub transforms the current compact Field Ops page from a small landing screen into a living **Underwatch Command Hub**. It connects character identity, faction work, missions, ciphers, forums, chat, Net Neighbors, archive systems, profile worlds, Signal Player, creator tools, volunteer pathways, and safe real-world readiness work into one coherent MMO experience.

The hub defines Hades Watch as a **nonviolent, community-centered, lore-driven web MMO**. It is not a combat arena. It is a game of identity, signal work, civic imagination, creative contribution, privacy literacy, archive recovery, mutual aid, puzzles, badges, old-web culture, and faction-aligned storytelling.

## Design Pillars

### 1. Identity Before Combat

The player builds an operative identity through callsign, avatar, profile world, relic zone, faction marks, badges, titles, contribution history, lore unlocks, and community presence. The hub makes identity meaningful without turning the game into a combat simulator.

Identity elements include:

- Character dossier
- Callsign
- Avatar builder
- Profile world
- Relic zone
- Badge case
- Faction affiliation
- Alliance mark
- Display title
- Lore unlock history
- Contribution history

### 2. Community as Gameplay

Community participation is not a side feature. It is part of the MMO loop. Forums, conversation threads, Net Neighbors, chat rooms, events, community builder submissions, contributor boards, and support pathways should all feel like game systems.

Community actions can unlock badges, reputation, titles, archive traces, and faction recognition when they are constructive, safe, and moderated.

### 3. Ciphers as Training

Ciphers are Oracular Circuit training exercises and Dead Index signal fragments. They teach puzzle-solving, pattern recognition, careful reading, symbolic decoding, and lore discovery. They must remain safe puzzle-game experiences, not real-world hacking instructions.

Cipher systems may include:

- First Cipher Set
- C1PH3R CR4K3R badge chain
- Hint reveals
- Dead Index lore fragments
- Oracular Circuit reputation
- Safe training puzzles
- Solved/unsolved status

### 4. Missions as Safe Real-World Readiness

Missions are not combat objectives. They reward care, preparation, documentation, repair, accessibility, mutual aid, privacy literacy, civic resilience, archiving, community support, creative morale, and nonviolent de-escalation education.

Mission systems should be framed as:

- Field Orders
- Underwatch Work
- Chthonic Tasks
- Dossier Actions
- Civic Readiness
- Signal Work

### 5. Archive as World Memory

The Archive remembers what the surface buries. It stores lore, public-interest resources, State of Affairs links, Ghost in Tech repos, unlocked fragments, faction records, community documents, and evidence-light mission outputs.

The Archive should feel like both a lore system and a practical memory system.

### 6. Old Web as Resistance Infrastructure

The old web is part of Hades Watch’s worldbuilding. Profile worlds, relic zones, 88x31 Net Neighbor banners, forums, personal pages, zine-style notes, and strange small web rituals are resistance infrastructure. They are how the Underwatch remains human inside a surveilled technocracy.

### 7. Privacy-Aware by Default

Gameplay should never pressure users to expose sensitive personal information. Proof packets, mission submissions, community posts, and profile customization should default to redaction, consent, minimal collection, and user control.

### 8. Nonviolent Progression

Progression must never reward violence, weapons, illegal hacking, harassment, doxxing, intimidation, sabotage, unsafe protest tactics, unsafe medical advice, or real-world harm. Players progress by building, caring, documenting, decoding, repairing, archiving, learning, and supporting.

## Roadmap Phases

### Phase 1: Hub Expansion MVP

Include:

- Expanded Field Ops page
- Cards/sections for all existing systems
- Links to existing routes
- Better empty states
- Badge display hooks
- Ciphers card
- Community card placeholder
- Archive card
- Avatar Forge card
- Signal Player status
- Support the Underwatch placeholder
- Full-page layout that uses available screen space better

Goal:

Make the MMO Hub feel like a real command dashboard even before every deeper system is complete.

### Phase 2: Community Systems

Include:

- Forums/conversation threads
- Community Builder submissions
- Net Neighbor integration
- Chat room cards
- Contributor board
- Moderation dashboards
- Forum categories
- Thread badges
- Community Builder badges

Goal:

Make community participation an MMO loop.

### Phase 3: Progression and Lore

Include:

- Badge milestones
- Faction reputation display
- Lore unlock previews
- Activity feed
- Event/transmission system
- Deeper profile world integration
- Hub personalization by faction
- Recent activity modules

Goal:

Make the hub feel alive and responsive to what the player has done.

### Phase 4: Advanced Systems

Include:

- Richer community events
- Encrypted-at-rest chat
- Creator marketplace/library if desired
- Seasonal transmissions
- Faction campaigns
- Accessibility-first contribution tooling
- Advanced archive curation
- Contributor role pathways
- Modular community governance tools

Goal:

Evolve Hades Watch from a themed site into a persistent, community-centered web MMO.

## Closing Frame

The expanded MMO Hub should make every approved user feel like they have entered a living Underwatch command layer. They should see a place for identity, faction work, ciphers, missions, community, archive memory, creator tools, and signals.

The hub should not ask players to become fighters. It should ask them to become signal readers, caretakers, archivists, builders, thread tenders, repair ghosts, cipher crackers, forum stewards, and strange old-web neighbors.

The guiding line:

```txt
The surface rewards spectacle. The Underwatch rewards signals that keep people alive, connected, and remembered.
```
